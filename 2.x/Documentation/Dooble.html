Introduction

An effective, elegant, simple, and zero-dependency Web browser. Dooble should be functional on any operating system where Qt 5.9.x is supported.

The source is readily available at https://github.com/textbrowser/dooble/tree/master/2.x.

[INSERT DOOBLE IMAGE]

 

Accepted / Blocked Domains

Dooble supports the accepting or blocking of specific domains. The Accepted / Blocked Domains window allows for the defining of domains. Domains are stored in the SQLite database file dooble_accepted_or_blocked_domains.db. An operating mode may also be prepared.

Note: The bundled Data directory contains the file dooble_blocked_domains.txt which may be imported.

[INSERT ACCEPTED / BLOCKED DOMAINS WINDOW IMAGE]

 

Certificate Exceptions

Web sites may raise SSL/TLS certificate errors. Some of these certificate errors may be overridden.

[INSERT CERTIFICATE ERROR EXCEPTION]

Once overridden, the Web site and the certificate error are recorded in the SQLite file dooble_certificate_exceptions.db and access to the specific Web site is allowed.

Overridden sites are presented in the Certificate Exceptions window. Within this window, exceptions may be revoked.

Cookies

The Cookies window depicts Dooble’s current cookies. The SQLite database dooble_cookies.db contains cookie data.

Domain Filter

May be used to display only the specified domain.

Purge Periodically

If enabled, unchecked domains will be purged every 15 seconds.

[INSERT COOKIES WINDOW IMAGE]

 

Downloads

Dooble supports the downloading of data. Active and previous downloads are depicted in the Downloads window. Active downloads may be canceled. Dooble does not provide a mechanism for restarting a canceled or interrupted download. Downloads data are stored in the SQLite database dooble_downloads.db.

[INSERT DOWNLOADS WINDOW IMAGE]

 

Favorites

Favorites supersede bookmarks. Included are various sort options. Favorites, along with history items, are stored in the SQLite database dooble_history.db database.

[INSERT FAVORITES DIALOG IMAGE]

History

This History window is a general-purpose container depicting Dooble’s browsing history. A simple search is included. Selected items may be removed via a context menu. The SQLite database file dooble_history.db contains history data, along with favorites data.

[INSERT HISTORY WINDOW IMAGE]

 

Private Windows

When browsing in private windows, Dooble does not save the following data:

  • •Cookies 

  • •History 

  • •Temporary Files 

  • •Visited Links 

While in private windows, Dooble does save:

  • •Downloads 

  • •Favorites

Security Considerations

  • •Dooble does not exercise secure memory. 

  • •The AES and Threefish implementations are not designed to be thread-safe. 

 

Settings

This section describes some of the areas of the Settings window.

Display

Pin Windows

Some support windows may be pinned. Pinning is the process of embedding support windows within a Dooble window.

Privacy

Credentials

Dooble provides a process of storing authentically-encrypted data in various databases. This process is completely optional.

Web

Local Storage

Required for HTML5 storage.

XSS Auditing

Per Qt’s documentation, XSS Auditing monitors load requests for cross-site scripting attempts. Suspicious scripts are blocked.

Note: Dooble does not remove the local WebEnginePersistentStorage directory during a reset. Please remove this directory after a reset completes.

Sources of Randomness

Dooble requires data streams of random data for a variety of cryptographic algorithms. This section briefly describes the sources of these data streams for various operating systems.

BSD

BSD-like systems acquire pseudo-random data from the /dev/random device.

Linux

Linux systems acquire pseudo-random data from the /dev/urandom device.

Windows

Please read https://msdn.microsoft.com/en-us/library/windows/desktop/aa379942(v=vs.85).aspx.

 

Supported Protocols

Dooble supports the FILE, FTP, GOPHER, and HTTP(S) protocols.
