Introduction

An effective, elegant, simple, and zero-dependency Web browser. Dooble should be functional on any operating system where Qt 5.9.x is supported.

[INSERT DOOBLE IMAGE]



Certificate Exceptions

Web sites may raise SSL/TLS certificate errors. Some of these certificate errors may be overridden.

[INSERT CERTIFICATE ERROR EXCEPTION]

Once overridden, the Web site and the certificate error are recorded in the SQLite dooble_certificate_exceptions.db database file and access to the specific Web site is allowed.

Overridden sites are presented in the Certificate Exceptions window. Within this window, exceptions may also be revoked.

Downloads

Dooble supports the downloading of data. Active and previous downloads are depicted in the Downloads window. Active downloads may be canceled. Dooble does not provide a mechanism for restarting a canceled or interrupted download.

Favorites

Favorites supersede bookmarks. Included are various sort options. Favorites, along with history items, are stored in the SQLite dooble_history.db database file.

History

This History window is a general-purpose container depicting the current history. A simple search is included. Selected items may be removed via a context menu. The SQLite database dooble_history.db file contains history data, along with favorites data.

[INSERT HISTORY WINDOW IMAGE]



Private Windows

When browsing in private windows, Dooble does not save the following data:

  • Cookies

  • History

  • Temporary Files

  • Visited Links

While in private windows, Dooble does save:

  • Downloads

  • Favorites

Security Considerations

  • Dooble does not exercise secure memory.

  • The AES and Threefish implementations are not designed to be thread-safe.



Sources of Randomness

Dooble requires data streams of random data for a variety of cryptographic algorithms. This section briefly describes the sources of these data streams on various operating systems.

BSD

BSD-like systems acquire pseudo-random data from the /dev/random device.

Linux

Linux systems acquire pseudo-random data from the /dev/urandom device.

Windows

Please read https://msdn.microsoft.com/en-us/library/windows/desktop/aa379942(v=vs.85).aspx.



Supported Protocols

Dooble supports the FILE, FTP, GOPHER, and HTTP(S) protocols.
